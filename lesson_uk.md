# Як створити сервіс: посібник для початківців

## Що таке сервіс?

**Сервіс** — це програма, яка працює у фоновому режимі на комп'ютері: запускається автоматично, продовжує працювати навіть коли ніхто не авторизований у системі, та виконує своє завдання непомітно. Приклади: вебсервер (обслуговує сайти), рушій бази даних або планувальник задач.

Уявіть сервіс як вимикач світла в коридорі готелю: вмикається, коли будівля «запускається», горить цілий день і не потребує, щоб хтось стояв поруч і тримав його увімкненим.

---

## Навіщо використовувати сервіс?

| Без сервісу | З сервісом |
|---|---|
| Програму потрібно запускати вручну | Запускається автоматично при завантаженні |
| Зупиняється при виході з системи | Продовжує працювати у фоні |
| Немає автоматичного перезапуску при збої | Може перезапустити себе після помилки |
| Важко керувати журналами | Журнали централізовані та організовані |

## Крок 1: Напишіть програму

Перш ніж створювати сервіс, потрібна програма для запуску. Використаємо простий Python-скрипт як приклад.

Створіть файл `my_service.py`:

```python
import time
import logging

# Налаштовуємо журналювання, щоб бачити, що робить сервіс
logging.basicConfig(
    filename='/var/log/my_service.log',
    level=logging.INFO,
    format='%(asctime)s - %(message)s'
)

def main():
    logging.info("Сервіс запущено!")
    while True:
        logging.info("Сервіс працює...")
        time.sleep(60)  # Чекаємо 60 секунд перед наступним записом

if __name__ == "__main__":
    main()
```

**Що робить цей код:** кожні 60 секунд скрипт записує повідомлення у файл журналу. Просто — але ідеально демонструє концепцію.

## Крок 2: Створіть файл сервісу systemd (Linux)

У Linux **systemd** — це стандартний інструмент для керування сервісами. Ви описуєте свій сервіс у звичайному текстовому файлі.

Створіть новий файл за шляхом `/etc/systemd/system/my_service.service`:

```ini
[Unit]
Description=Мій перший сервіс
After=network.target

[Service]
ExecStart=/usr/bin/python3 /home/youruser/my_service.py
Restart=always
User=youruser

[Install]
WantedBy=multi-user.target
```

**Розберемо по частинах:**

- `Description` — зрозуміла назва вашого сервісу
- `After=network.target` — чекати готовності мережі перед запуском
- `ExecStart` — точна команда, яка запускає вашу програму
- `Restart=always` — якщо програма впаде, перезапустити автоматично
- `User` — від імені якого користувача системи запускати сервіс
- `WantedBy=multi-user.target` — запускати цей сервіс при звичайному завантаженні системи

## Крок 3: Увімкніть та запустіть сервіс

Відкрийте термінал і виконайте ці команди по черзі:

```bash
# 1. Повідомте systemd про нові файли сервісів
sudo systemctl daemon-reload

# 2. Увімкніть сервіс для автозапуску при кожному завантаженні
sudo systemctl enable my_service

# 3. Запустіть сервіс прямо зараз
sudo systemctl start my_service
```

---

## Крок 4: Перевірте, що сервіс працює

```bash
# Переглянути статус сервісу
sudo systemctl status my_service
```

Ви побачите щось на зразок:

```
● my_service.service - Мій перший сервіс
     Loaded: loaded (/etc/systemd/system/my_service.service; enabled)
     Active: active (running) since ...
```

**«active (running)»** означає, що ваш сервіс живий і працює. 🎉

---

## Крок 5: Перегляньте журнали

```bash
# Переглянути останні записи журналу сервісу
sudo journalctl -u my_service -n 50
```

Ця команда показує останні 50 рядків виводу. Це ваше вікно в те, що сервіс насправді робить.

---

## Шпаргалка з основних команд

| Команда | Що робить |
|---|---|
| `systemctl start my_service` | Запустити сервіс |
| `systemctl stop my_service` | Зупинити сервіс |
| `systemctl restart my_service` | Перезапустити сервіс |
| `systemctl status my_service` | Перевірити статус |
| `systemctl enable my_service` | Автозапуск при завантаженні |
| `systemctl disable my_service` | Прибрати автозапуск |
| `journalctl -u my_service` | Переглянути журнали |

## Поради щодо усунення проблем

1. **Сервіс не запускається?** Виконайте `systemctl status my_service` — зазвичай там є точна помилка.
2. **Помилки доступу?** Перевірте, що користувач, вказаний у `User=`, має доступ до скрипту та файлу журналу.
3. **Зміни не застосовуються?** Завжди виконуйте `sudo systemctl daemon-reload` після редагування файлу `.service`.
4. **Неправильний шлях до скрипту?** Перевірте повний шлях у `ExecStart` — використайте `which python3`, щоб знайти правильний шлях до Python.

## Підсумок

Створення сервісу складається лише з трьох речей:
1. **Програма** для запуску
2. **Файл опису сервісу**, який пояснює системі, як її запускати
3. **Кілька команд** для реєстрації та запуску

Коли сервіс запущено, система керує ним сама: перезапускає при збоях, запускає при завантаженні, веде журнали. У цьому і полягає сила перетворення скрипту на сервіс.
